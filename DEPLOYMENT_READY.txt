## üöÄ Ready to Deploy!

### **QUICK START (Choose One)**

#### **Option 1: Cloud-Init (Fastest - Recommended)**
1. Create Vultr account: https://www.vultr.com/
2. Click "Deploy New Server"
3. Select: Ubuntu 22.04 LTS, 2GB RAM
4. Scroll down to "User Data" section
5. **Copy the ENTIRE contents of `vultr-cloud-init.sh`**
6. Paste into the User Data text box
7. Click "Deploy Now"
8. Wait 5-10 minutes ‚è≥
9. SSH to your server: `ssh root@YOUR_SERVER_IP`
10. Configure: `nano /opt/financial-project/.env`
    - Add: `OPENAI_API_KEY=sk-your-key-here`
11. Restart: `docker compose restart`
12. Access at: `https://YOUR_SERVER_IP`

#### **Option 2: Manual Script Deployment**
```bash
# SSH to your Vultr server
ssh root@YOUR_SERVER_IP

# Run deployment script
bash <(curl -s https://raw.githubusercontent.com/vushakolaPhanindra/Financial_Project/main/deploy.sh)

# Configure and restart
nano /opt/financial-project/.env
docker compose restart
```

#### **Option 3: Docker Compose (Local/Manual)**
```bash
# Clone and deploy
git clone https://github.com/vushakolaPhanindra/Financial_Project.git
cd Financial_Project
cp .env.example .env
nano .env  # Add OPENAI_API_KEY
docker compose up -d
```

## üÜò Quick Troubleshooting

**Container won't start?**
```bash
docker compose logs
docker compose build --no-cache
docker compose up -d
```

**Can't connect to website?**
```bash
sudo ufw allow 80,443/tcp
docker compose logs nginx
```

**API returning errors?**
```bash
docker compose logs backend
docker compose restart backend
```

**Out of disk space?**
```bash
docker system prune -a
df -h
```
